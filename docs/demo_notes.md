# Demo 讲解笔记

## 这个项目解决什么问题

项目模拟一个简单的网格寻路任务：智能体从左上角起点 `S` 出发，目标是到达右下角目标点 `G`，中间的 `#` 是障碍物。智能体不知道最优路线，需要通过不断试错学习“哪些状态下应该往哪个方向走”。

## 环境设计

- 状态：`6 x 6` 网格被展开成长度为 `36` 的向量。
- 动作：上、下、左、右四个离散动作。
- 奖励：每走一步有轻微惩罚，撞墙或撞障碍物有较大惩罚，到达目标点获得正奖励。
- 终止：到达目标点，或者超过最大步数。

## DQN 设计

DQN 用神经网络估计每个动作的 Q 值。输入是当前网格状态，输出是四个动作对应的价值。

训练中包含几个关键机制：

- epsilon-greedy：前期多探索，后期多利用已经学到的策略。
- replay buffer：把历史经验存起来，随机抽样训练，减少样本相关性。
- target network：使用一个延迟更新的目标网络，让训练目标更稳定。
- SmoothL1Loss：比普通 MSE 对异常值更稳健。

## 运行流程

1. `train.py` 创建环境和智能体。
2. 智能体根据状态选择动作。
3. 环境返回下一个状态、奖励、是否结束。
4. 经验进入 replay buffer。
5. DQN 从 replay buffer 抽样学习。
6. 训练结束后保存模型和曲线。
7. `test.py` 加载模型，用贪心策略测试路线。

## 面试/简历可以怎么讲

可以这样概括：

> 我实现了一个基于 PyTorch 的 GridWorld-DQN 强化学习 demo。项目里我自定义了网格环境、奖励函数和状态表示，并实现了 DQN 智能体，包括经验回放、epsilon-greedy 探索策略和目标网络更新。训练过程中会记录奖励、成功率和 loss，并输出训练曲线和测试轨迹，方便分析智能体是否真正学会避障到达目标。

如果被问到“为什么需要 replay buffer”，可以回答：

> 强化学习中的连续样本相关性很强，直接按时间顺序训练容易不稳定。replay buffer 把经验存起来后随机采样，可以打乱样本顺序，提高训练稳定性和样本利用率。

如果被问到“为什么需要 target network”，可以回答：

> DQN 的目标值本身也来自神经网络，如果每一步都用同一个快速变化的网络估计目标，训练会比较震荡。target network 延迟同步参数，让目标值相对稳定。
